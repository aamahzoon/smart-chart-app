# راهنمای تبدیل برنامه به فایل exe مستقل (ویندوز)

این برنامه یک اپ Streamlit است. Streamlit ذاتاً یک وب‌سرور محلی راه می‌اندازد و در
مرورگر نمایش داده می‌شود؛ یعنی حتی بعد از تبدیل به exe هم با اجرا شدنش یک پنجرهٔ
مرورگر باز می‌شود (این طبیعی و مورد انتظار است، نه یک باگ).

## فایل‌های لازم (همه در این پوشه آماده‌اند)
- `app.py` — خود برنامه
- `run_app.py` — لانچری که Streamlit را از داخل پایتون اجرا می‌کند (چون PyInstaller
  نمی‌تواند دستور `streamlit run` را مستقیم فریز کند)
- `SmartChartExplorer.spec` — فایل تنظیمات PyInstaller
- `requirements.txt` — لیست پکیج‌های لازم

## مراحل ساخت exe

### ۱. آماده‌سازی محیط (روی ویندوز)
```bat
python -m venv build_env
build_env\Scripts\activate
pip install -r requirements.txt
```

### ۲. تست اجرای مستقیم (قبل از build، برای اطمینان از سالم بودن همه‌چیز)
```bat
python run_app.py
```
باید یک پنجرهٔ مرورگر با برنامه باز شود. اگر باز شد، مرحلهٔ بعد را انجام بده.

### ۳. ساخت exe با PyInstaller
```bat
pyinstaller SmartChartExplorer.spec
```
این فرمان چند دقیقه طول می‌کشد (چون Streamlit، Plotly، SciPy و Statsmodels حجیم‌اند).
خروجی نهایی در مسیر زیر قرار می‌گیرد:
```
dist\SmartChartExplorer\SmartChartExplorer.exe
```

### ۴. اجرای exe
کل پوشهٔ `dist\SmartChartExplorer` را (نه فقط فایل exe را) به هر جایی که می‌خواهی
منتقل کن و `SmartChartExplorer.exe` را اجرا کن. یک پنجرهٔ کنسول کوتاه باز می‌شود و
بلافاصله مرورگر پیش‌فرض سیستم با آدرس `http://localhost:8501` باز خواهد شد.

> ⚠️ نکته: چون خروجی PyInstaller در حالت `COLLECT` (نه `--onefile`) ساخته شده، همهٔ
> فایل‌های کنار exe (پوشهٔ `_internal` و غیره) باید حتماً همراه exe منتقل شوند؛
> exe به‌تنهایی کار نمی‌کند. این روش پایدارتر از `--onefile` است چون فایل‌های
> استاتیک Streamlit (که در زمان اجرا لازم‌اند) داخل exe فشرده نمی‌شوند.

## نکات مهم و رفع اشکال رایج

- **صفحه در مرورگر باز نمی‌شود ولی exe اجرا شده**: چند ثانیه صبر کن (بار اول کمی
  طول می‌کشد)، یا خودت آدرس `http://localhost:8501` را در مرورگر باز کن.

- **خطای `ModuleNotFoundError` هنگام اجرای exe**: یعنی یک پکیج به‌درستی جمع‌آوری
  نشده. نام پکیج را به لیست `packages_to_collect` در `SmartChartExplorer.spec`
  اضافه کن و دوباره `pyinstaller SmartChartExplorer.spec` را اجرا کن.

- **آنتی‌ویروس فایل exe را مسدود می‌کند**: این مشکل رایج و شناخته‌شدهٔ فایل‌های
  ساخته‌شده با PyInstaller است (نه ویروس واقعی)؛ یا از حالت `--onedir` (پیش‌فرض
  همین spec) استفاده کن که هشدار کمتری می‌گیرد، یا فایل را روی سیستم مقصد
  Whitelist کن.

- **کوچک‌تر کردن حجم خروجی**: با نصب `upx` (https://upx.github.io) و قرار دادن آن
  در PATH سیستم، PyInstaller به‌صورت خودکار از آن برای فشرده‌سازی باینری‌ها
  استفاده می‌کند (در spec فعلی `upx=True` تنظیم شده).

- **اجرا روی سیستم‌های بدون اینترنت**: توجه داشته باش که فونت Vazirmatn و
  کتابخانهٔ Plotly.js در گزارش HTML خروجی (فایلی که کاربر دانلود می‌کند) از CDN
  بارگذاری می‌شوند، پس آن فایل HTML گزارش برای نمایش صحیح به اینترنت نیاز دارد؛
  اما خود برنامهٔ exe (رابط Streamlit) کاملاً آفلاین اجرا می‌شود.

- **ساخت نسخهٔ macOS/Linux**: همین `run_app.py` و `requirements.txt` روی
  macOS/Linux هم با همان فرمان `pyinstaller SmartChartExplorer.spec` کار می‌کند؛
  فقط باید روی همان سیستم‌عامل ساخته شود (PyInstaller قابلیت cross-compile ندارد).

## چرا کد app.py نیازی به تغییر نداشت؟
برنامه از هیچ فایل خارجی یا مسیر نسبی (`__file__`, مسیرهای دیسک و…) استفاده
نمی‌کند و تمام صفحه‌آرایی، فونت و کتابخانه‌هایش یا از طریق pip نصب می‌شوند یا از
CDN در خروجی HTML لود می‌شوند؛ بنابراین برای فریز شدن با PyInstaller آماده بود و
فقط یک لانچر (`run_app.py`) لازم داشت.
